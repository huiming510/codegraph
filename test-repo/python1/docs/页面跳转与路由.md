# 页面跳转与路由文档

本文档描述前端路由结构、页面与路径对应、鉴权与跳转规则、典型业务流程，便于前后端对齐与新人理解导航逻辑。

---

## 1. 路由模式

| 项目   | 说明                                         |
| ------ | -------------------------------------------- |
| 模式   | 默认 **hash**（`VITE_APP_ROUTER_MODE=hash`） |
| 示例   | 登录页：`http://localhost:5174/#/login`      |
| 根路径 | `/` 重定向到 `/chat`                         |

---

## 2. 路由分类

### 2.1 静态路由

| path    | name   | 组件                  | 说明                      |
| ------- | ------ | --------------------- | ------------------------- |
| /       | -      | redirect              | 重定向到 /chat            |
| /login  | login  | views/login/index.vue | 登录页                    |
| /layout | layout | layouts/index.vue     | 主布局，redirect 到 /chat |

### 2.2 错误页路由

| path             | name | 组件               | 说明                         |
| ---------------- | ---- | ------------------ | ---------------------------- |
| /403             | 403  | ErrorPages/403.vue | 无权限                       |
| /404             | 404  | ErrorPages/404.vue | 页面不存在                   |
| /500             | 500  | ErrorPages/500.vue | 网络异常（白名单，无需登录） |
| /:pathMatch(._)_ | -    | ErrorPages/404.vue | 兜底 404                     |

### 2.3 动态路由（由后端菜单接口返回）

登录后调用 `GET /api/auth/menu`，根据返回的菜单结构通过 `router.addRoute` 注入。组件路径约定：`component: "/xxx/yyy/index"` 对应 `@/views/xxx/yyy/index.vue`。

## 4. 鉴权与跳转规则

### 4.1 路由守卫（router.beforeEach）

| 条件              | 行为                                                                  |
| ----------------- | --------------------------------------------------------------------- |
| 访问 /login       | 有 token → 留在当前页；无 token → 放行                                |
| 访问白名单路径    | 直接放行（当前仅 `/500`）                                             |
| 无 token          | 重定向到 `/login`，replace                                            |
| 有 token 但无菜单 | 请求 `getMenuList`，注入动态路由，再 `next({ ...to, replace: true })` |
| 有 token 且有菜单 | 放行                                                                  |

### 4.2 白名单

| 路径 | 说明                       |
| ---- | -------------------------- |
| /500 | 网络异常页，无需登录可访问 |

### 4.3 401 处理

- 请求拦截器：响应 401 时清除 token，跳转登录页。
- 详见 `src/frontend/src/api/index.js` 与 `config/request.js`。

## 5. 典型业务流程与跳转

### 5.1 登录流程

```
未登录访问任意需鉴权路径
    → 重定向 /login
    → 输入账号密码，POST /api/auth/login
    → 成功：存 token，请求 /api/auth/menu
    → 注入动态路由
    → 跳转 /chat（或 from.fullPath）
```

### 5.2 知识库流程

```
侧栏「知识库」→ /knowledge/knowledgeMgt（知识库管理）
    → 点击某知识库 → /knowledge/documentMgt（文档管理，或带 kb 参数）
    → 上传文档 → 可能跳转 /knowledge/upload/
```

### 5.3 对话流程

```
侧栏「智能问答」→ /chat/list
    → 新建/选择会话 → /chat/detail（可能带 session 参数）
```

### 5.4 智能检索流程

```
侧栏「智能检索」→ /query/rag
    → 提交检索 → /query/detail（检索结果详情）
```

### 5.5 系统设置（仅 admin）

```
右上角用户下拉 → 系统设置 → /system/userMgt（或 llmConfig、esConfig、logs）
```

## 6. 菜单与权限

| 角色  | 可见菜单                                                         |
| ----- | ---------------------------------------------------------------- |
| admin | 首页、智能问答、工作流、知识库、智能检索、文档管理、**系统设置** |
| user  | 首页、智能问答、工作流、知识库、智能检索、文档管理               |

菜单结构由后端 `database/crud.get_full_menu_list()` 硬编码返回，按角色过滤。详见 [前后端可扩展性分析](./前后端可扩展性分析.md)。
