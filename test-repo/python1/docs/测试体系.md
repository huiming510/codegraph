# 测试体系

本文说明项目前后端测试的约定、运行方式、目录结构与扩展方式，便于日常疏通测试与后续接入 CI。

---

## 一、总体约定

| 维度 | 后端 | 前端 |
|------|------|------|
| 框架 | pytest | Vitest + @vue/test-utils |
| 运行入口 | `pytest src/backend/tests` 或 `scripts/run_tests.ps1` | `npm run test` / `npm run test:run`（在 `src/frontend` 下） |
| 定位 | 疏通 / 冒烟优先：接口 200、主路径可跑通 | 疏通优先：工具/常量可测、关键组件可挂载 |
| 覆盖率 | 可选：`pytest --cov=src/backend` | 可选：`npm run test:coverage` |

**用例依据**：接口测试的断言（如响应 `code`、`data` 结构）必须以 [API 文档 1.2 节](./API文档.md#12-统一响应体测试用例必须依此断言) 为准。文档不全时生成的用例易出错；详见 [测试策略与用例生成 - 4.2 节](./测试策略与用例生成.md#42-辅助材料清单生成前建议准备)。

**CI 覆盖率与门禁**（已接入 GitLab CI，见 [GitLab分支与质量门禁](./GitLab分支与质量门禁.md)）：
- **后端**：`backend-test` 会采集 coverage（pytest-cov），产出 `coverage.xml`（Cobertura）。门禁可选：在 GitLab CI 变量中设置 **`COVERAGE_BACKEND_MIN`**（如 `60`），则覆盖率低于该百分比时 job 失败。
- **前端**：`frontend-test` 使用 `npm run test:coverage`，产物在 `src/frontend/coverage/`。若需门禁，可在 `src/frontend/vite.config.js` 的 `test.coverage` 中增加 `thresholds: { lines: 40 }`（未达标时 test:coverage 失败）。

## 二、后端测试（FastAPI）

### 2.1 依赖安装

在项目根目录（或 backend 所在环境）安装测试依赖：

```bash
# 使用 pip（项目根）
pip install pytest pytest-asyncio httpx

# 或使用 uv / 可选依赖
uv pip install -e ".[test]"
```

### 2.2 目录与约定

- **目录**：`src/backend/tests/`
- **配置**：根目录 `pyproject.toml` 中 `[tool.pytest.ini_options]`
- **公共 fixture**：`tests/conftest.py`（测试库路径、client、auth_headers）
- **用例命名**：`test_*.py`，按模块分文件

### 2.3 运行方式

在项目根目录：

```powershell
$env:PYTHONPATH = ".\src"
pytest src/backend/tests -v
# 或
.\scripts\run_tests.ps1
```

### 2.4 用例与测试数据

- **健康与认证**：`test_health_and_auth.py`、`test_auth_api.py`（根路径、登录/注册、userinfo、登出、菜单、改密）。
- **用户/权限/日志**：`test_users_api.py`、`test_permissions_api.py`、`test_logs_api.py`（管理员与普通用户权限、列表与筛选）。
- **知识库/文档/RAG**：`test_knowledge_bases_api.py`、`test_documents_api.py`、`test_rag_api.py`（CRUD、分页、会话与检索）。
- **配置**：`test_config_api.py`（当前配置、ES、语言）。
- **测试数据**：见 `src/backend/tests/fixtures/README.md`。conftest 提供 `auth_headers`（admin）、`test_user_headers`（testuser）、`extra_kb_id`（额外知识库）；更多数据可通过 API 在用例中创建。

## 三、前端测试（Vue 3 + Vite）

### 3.1 依赖

`src/frontend` 下 `npm install`，已含 vitest、@vue/test-utils、jsdom、@vitest/coverage-v8。

### 3.2 目录与约定

- **配置**：`vite.config.js` 中 `test` 块
- **用例位置**：`src/**/*.{spec,test}.{js,ts}`，如 `src/__tests__/example.spec.js`

### 3.3 运行方式

在 `src/frontend` 下：

```bash
npm run test
npm run test:run
npm run test:coverage
```

### 3.4 前端测试分布

| 类型 | 位置 | 说明 |
|------|------|------|
| 常量/工具 | `src/__tests__/example.spec.js` | 路由常量、占位组件 |
| API 模块 | `src/api/modules/__tests__/*.spec.js` | 登录等接口封装（mock request） |
| 组件 | `src/views/**/__tests__/*.spec.js` | 如 LoginForm 挂载与表单项 |

### 3.5 E2E 测试（Playwright）

已接入 **Playwright**，在 `src/frontend` 下：

- **安装**：`npm install -D @playwright/test`，首次需 `npx playwright install chromium`。
- **运行**：`npm run test:e2e`（会自动启动 dev 服务器并跑用例）；`npm run test:e2e:ui` 可打开 UI 模式。
- **用例位置**：`src/frontend/e2e/*.spec.js`，当前含 `login.spec.js`：
  - 登录页：表单可见、按钮可点击、完整登录流程（需后端）、错误密码提示
  - 鉴权：未登录访问 /、/chat、/knowledge 跳转登录
  - 登录后主流程：知识库管理页、智能检索页（需后端）
  - 错误页：403、404、500 文案可见
- **配置**：`src/frontend/playwright.config.js`，默认使用 Chromium，`baseURL` 为 `http://localhost:5174`（与 Vite 默认端口一致）。
- **截图**：用例失败时自动截图到 `src/frontend/test-results/`；另有「登录页截图」用例每次成功会保存 `test-results/screenshots/login-page.png`。`test-results/` 已加入 .gitignore。

## 四、统一运行脚本

项目根目录执行：`.\scripts\run_tests.ps1`，会依次运行后端 pytest 与前端 test:run。
