# RAG 知识库 - 开发框架说明

本文档描述 **backend** 与 **frontend** 的整体架构与目录约定，其他仓库目录（如 `src/preprocess`、`src/index` 等）为既有能力，开发时可按需调用，不作为日常开发主目录。

---

## 一、整体架构

```
linkrag/
├── src/
│   ├── backend/          ← 后端开发目录（FastAPI）
│   └── frontend/         ← 前端开发目录（Vue3 + Vite）
├── docs/                 ← 文档（本说明、接口契约等）
└── ...
```

- **后端**：`src/backend`，Python 3.12+，FastAPI + SQLAlchemy（异步）+ SQLite，JWT 认证，可选 Elasticsearch。
- **前端**：`src/frontend`，Vue 3 + Vite，Pinia，Vue Router，Ant Design Vue，axios 封装。
- **前后端约定**：接口统一前缀 `/api`，响应体统一为 `{ code, data, msg }`（成功时 `code` 为 0 或 200，前端取 `data`）。

---

## 二、Backend 结构（src/backend）

| 目录/文件 | 说明 |
|-----------|------|
| `main.py` | 应用入口：FastAPI 实例、生命周期、全局异常处理、挂载路由 |
| `config.py` | 应用配置（Pydantic Settings，环境变量 / .env） |
| `database/` | 数据库：`models.py` 表结构，`crud.py` 增删改查，`connection.py` 异步会话 |
| `routers/` | 业务路由：按模块拆分（auth、users、knowledge_bases、documents、rag 等） |
| `routers/common.py` | 统一成功响应：`success_response(data=None, msg="success")` → `{ code: 0, data, msg }` |
| `routers/deps.py` | 依赖：JWT、`get_current_user`、`require_user`、`require_admin` |
| `llm/` | LLM 抽象与多厂商实现（OpenAI、Azure、Qwen、智谱、Ollama、Mock 等） |
| `logger/` | 日志配置与请求日志中间件 |
| `server/` | 解析与 RAG 服务：文件解析 worker、ES 写入、RAG 检索等（依赖项目 `preprocess`、`index`） |

### 启动方式

需设置 `PYTHONPATH` 包含项目 `src` 目录，并在 **src/backend** 下启动：

```powershell
$env:PYTHONPATH = "d:\ai-rag\linkrag\src"
cd d:\ai-rag\linkrag\src\backend
..\..\.venv\Scripts\python.exe -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

接口文档：`http://localhost:8000/docs`。

### 路由与响应约定

- 路由前缀示例：`/api/knowledge-bases`、`/api/auth/login` 等，在 `main.py` 中通过 `include_router(router)` 挂载。
- 成功：使用 `from routers.common import success_response`，返回 `success_response(data=...)`。
- 错误：抛出 `HTTPException`，由 `main.py` 统一处理为 `{ code: status_code, data: null, msg }`。

## 三、Frontend 结构（src/frontend）

| 目录/文件 | 说明 |
|-----------|------|
| `src/main.js` | 应用入口 |
| `src/App.vue` | 根组件 |
| `src/api/index.js` | axios 封装实例（baseURL、拦截器、统一处理 `{ code, data, msg }`） |
| `src/api/modules/*.js` | 按业务模块的 API 方法（如 `knowledgeBase.js`、`document.js`、`login.js`） |
| `src/config/request.js` | 请求常量（如超时、状态码） |
| `src/router/` | 路由：`index.js` 创建路由与守卫，`modules/` 下静态/动态路由 |
| `src/stores/` | Pinia 状态：`auth`、`user`、`global` 等 |
| `src/views/` | 页面级组件（知识库、文档、对话、系统配置等） |
| `src/layouts/` | 布局（含侧栏、顶栏等） |
| `src/components/` | 通用 UI 组件（Loading、ErrorPages、SearchBar 等） |
| `src/constants/` | 常量（路由白名单、应用名等） |
| `src/styles/` | 全局样式（reset、common、pixel 等） |
| `src/utils/` | 工具函数 |
| `.env.development` | 开发环境变量（如 `VITE_APP_API_BASE_URL=/api`，端口、代理等） |
| `vite.config.js` | Vite 配置；代理将 `/api` 转发到后端（需与后端端口一致） |

### 请求与 API 约定

- 所有请求经 `src/api/index.js` 封装的 axios 实例，baseURL 为 `/api`（开发时由 Vite 代理到后端）。
- 响应拦截器：若 `data.code === 0` 或 `200` 则返回 `data.data`，否则提示 `data.msg` 并 reject。
- 新增接口时：在 `src/api/modules/` 下对应模块增加方法，路径与后端一致（不含 `/api` 前缀，因 baseURL 已带）。

### 路由与权限

- 登录态：由 store（如 `user`）存 token，请求头带 `Authorization: Bearer <token>`；401 时跳转登录。
- 动态路由：根据权限/菜单接口拉取后，通过 `router.addRoute` 注入；布局名 `layout`，全屏页用 `meta.isFull`。

## 四、前后端联调要点

1. **端口**：后端默认 8000；前端 Vite 开发端口见 `.env.development`（如 5174），代理目标需与后端一致（当前 vite 中为 8003，若后端用 8000 需改为 8000）。
2. **统一响应**：后端统一用 `success_response` 与 `HTTPException`，前端按 `{ code, data, msg }` 处理。
3. **认证**：登录接口返回 token，前端存 store 并写入请求头；需要登录的接口使用 `Depends(require_user)` 或 `require_admin`。

---

## 五、开发规约（AI 生成代码时）

- 仅将 **src/backend** 与 **src/frontend** 视为本系统开发目录；其他目录为既有能力，不随意改结构。
- 后端：新路由放在 `routers/` 下，使用 `success_response` 与 `deps` 中的认证依赖；不破坏 `main.py` 的异常处理与 CORS 配置。
- 前端：新 API 放在 `api/modules` 对应模块，新页面放在 `views/` 下；使用现有 axios 实例与路由/权限体系。
- 详细规约见项目根目录 **`.cursor/rules/`** 下的规则文件，供 AI 与开发者统一遵守。
