# 用户管理中心 - 修改总结

本文档汇总**用户管理中心**相关的前端界面、后端服务接口、数据库及访问服务、页面/功能跳转逻辑（入参）。

---

## 1. 前端界面

### 1.1 页面位置与入口

| 项目 | 说明 |
|------|------|
| **页面路径** | `src/frontend/src/views/system/userMgt/index.vue` |
| **路由 path** | `/system/userMgt` |
| **菜单归属** | 系统设置 → 用户管理（仅 **管理员 admin** 可见，由后端 `GET /api/auth/menu` 按角色过滤） |

### 1.2 页面功能

- **页面标题**：用户管理；副标题：管理系统用户与知识库角色（管理员 / 开发人员 / 用户）。
- **统计卡片**：总用户数、活跃用户、管理员数、开发人员数、用户(guest)数（由列表前端计算）。
- **用户列表表格**：列包括用户(昵称+@用户名)、邮箱、角色、状态(正常/禁用)、最后登录、操作。
- **操作**：编辑、重置密码（固定为 123456）、删除（管理员账号不可删）。
- **添加/编辑弹窗**：用户名、密码(仅添加时)、昵称、邮箱、角色(管理员/开发人员/用户)、状态(仅编辑时)。

### 1.3 前端 API 模块

**文件**：`src/frontend/src/api/modules/user.js`

| 方法 | 说明 | 对应后端 |
|------|------|----------|
| `getUserListApi(params)` | 获取用户列表 | GET /api/users |
| `createUserApi(params)` | 创建用户 | POST /api/users |
| `updateUserApi(userId, params)` | 更新用户 | PUT /api/users/{user_id} |
| `deleteUserApi(userId)` | 删除用户 | DELETE /api/users/{user_id} |
| `resetPasswordApi(userId, password)` | 管理员重置密码 | PUT /api/users/{user_id}/password |

---

## 2. 后端服务接口

### 2.1 路由模块

**文件**：`src/backend/routers/users.py`  
**前缀**：`/api/users`  
**权限**：所有接口均依赖 `Depends(require_admin)`，仅管理员可访问。

### 2.2 接口列表

| 方法 | 路径 | 说明 | 主要入参 / Body |
|------|------|------|------------------|
| GET | /api/users | 用户列表（支持分页、按角色筛选） | Query: `skip`, `limit`, `role` |
| POST | /api/users | 创建用户 | Body: `username`, `password`, `email?`, `nickname?`, `role?`（默认 user） |
| PUT | /api/users/{user_id} | 更新用户 | Body: `nickname?`, `email?`, `role?`, `is_active?` |
| PUT | /api/users/{user_id}/password | 管理员重置密码 | Body: `{ "password": "123456" }` |
| DELETE | /api/users/{user_id} | 删除用户 | Path: `user_id`（禁止删除 role=admin 的用户） |

### 2.3 角色约定（与鉴权中心一致）

- **admin**：管理员  
- **user**：开发人员  
- **guest**：用户（仅检索、聊天）

## 3. 数据库及访问服务

### 3.1 表：users

**文档**：`docs/数据库设计.md` 第 3.1 节  
**模型**：`src/backend/database/models.py` 中 `User` 类

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| username | VARCHAR(50) | 用户名，唯一 |
| password_hash | VARCHAR(128) | 密码哈希 |
| email | VARCHAR(100) | 邮箱，可空 |
| nickname | VARCHAR(50) | 昵称，默认「用户」 |
| avatar | VARCHAR(255) | 头像 URL，可空 |
| role | VARCHAR(20) | admin / user / guest，默认 user |
| is_active | BOOLEAN | 是否启用，默认 TRUE |
| last_login | DATETIME | 最后登录时间，可空 |
| created_at / updated_at | DATETIME | 创建/更新时间 |

**索引**：`idx_users_role`、`idx_users_active`。

### 3.2 CRUD 与访问服务

**文件**：`src/backend/database/crud.py`

| 方法 | 说明 |
|------|------|
| `hash_password` / `verify_password` | 密码哈希与校验 |
| `create_user(db, username, password, email?, nickname?, role?)` | 创建用户 |
| `get_user(db, user_id)` | 按 ID 获取用户 |
| `get_user_by_username(db, username)` | 按用户名获取用户 |
| `get_users(db, skip?, limit?, role?, is_active?)` | 用户列表（分页、筛选） |
| `update_user(db, user_id, **kwargs)` | 更新用户字段 |
| `update_user_password(db, user_id, new_password)` | 更新密码 |
| `update_user_last_login(db, user_id)` | 更新最后登录时间 |
| `delete_user(db, user_id)` | 删除用户 |
| `count_users(db, role?)` | 统计用户数（可选按角色） |
| `authenticate_user(db, username, password)` | 登录校验（供 auth 使用） |

菜单与角色：`get_full_menu_list()`、`get_menus_for_role(role)` 在 crud 中定义，供 `GET /api/auth/menu` 使用；**仅 admin** 角色会包含「系统设置」下的「用户管理」菜单。

## 4. 页面 / 功能跳转逻辑（入参）

### 4.1 进入用户管理页

| 方式 | 说明 | 入参 |
|------|------|------|
| **菜单** | 登录后，仅 **admin** 可见「系统设置 → 用户管理」，点击进入 | 无；路由 path 固定为 `/system/userMgt` |
| **直接 URL** | 浏览器访问 `/system/userMgt` | 无 path 参数；若未登录或非 admin，会因菜单/权限被拦截或重定向 |

当前实现下，用户管理页**无路由 query 参数**（如 `?role=admin`）；列表筛选通过接口的 `skip/limit/role` 实现，前端表格为前端内存分页（pageSize: 10）。

### 4.2 页面内操作与入参

| 操作 | 触发方式 | 入参 / 说明 |
|------|----------|-------------|
| **加载列表** | 页面 `onMounted` 调用 `loadUsers()` | 无；请求 `GET /api/users`，前端取 `data.users` |
| **添加用户** | 点击「添加用户」→ 打开弹窗 → 确定 | 表单：username, password, nickname, email, role |
| **编辑用户** | 点击某一行的「编辑」→ 打开弹窗 → 确定 | 行数据 `record` 作为 `editingUser`；提交时 `updateUserApi(record.id, { nickname, email, role, is_active })` |
| **重置密码** | 点击「重置密码」→ 二次确认 | `resetPasswordApi(record.id, "123456")`，无前端入参 |
| **删除用户** | 点击「删除」→ 二次确认 | `deleteUserApi(record.id)`；若 `record.role === 'admin'` 按钮禁用 |

### 4.3 与认证/菜单的关系

- **登录**：`POST /api/auth/login`（见 `routers/auth.py`），成功后前端存 token，并请求 `GET /api/auth/menu` 获取菜单。
- **菜单**：`GET /api/auth/menu` 返回 `crud.get_menus_for_role(user.role)`；仅当 `role === 'admin'` 时包含系统设置及子菜单「用户管理」（path: `/system/userMgt`）。
- **鉴权**：用户管理相关接口均使用 `require_admin`，非管理员调用会返回 401/403。

---

## 5. 小结

| 层级 | 内容 |
|------|------|
| **前端** | 页面 `views/system/userMgt/index.vue`，API 封装在 `api/modules/user.js`，路由 path `/system/userMgt`，无路由入参。 |
| **后端** | `routers/users.py` 提供用户列表/创建/更新/重置密码/删除，全部需 admin；角色为 admin / user / guest。 |
| **DB 与访问** | 表 `users`，模型 `User`；CRUD 与鉴权在 `database/crud.py`，菜单在 crud 中按角色过滤，仅 admin 可见用户管理。 |
| **跳转与入参** | 仅通过「系统设置 → 用户管理」或直接访问 `/system/userMgt`，无 path/query 入参；表格内操作以行数据 `record.id` 等作为接口入参。 |
