# 数据库管理与迁移

本文档约定本项目数据库文件管理方式，并提供 SQLite 与 PostgreSQL 的切换/迁移流程。

---

## 1. 本地数据库文件管理

- 本地开发默认可使用 `src/backend/linkrag.db`（SQLite）。
- `linkrag.db` 不纳入 Git 管理（仓库 `.gitignore` 已忽略 `*.db` / `*.sqlite*`）。
- 结构演进通过迁移脚本和文档管理，不通过提交数据库文件。

---

## 2. 连接配置

后端优先读取 `DATABASE_URL`，未设置时回退到 `SQLITE_DATABASE`。

### 2.1 SQLite

```dotenv
DATABASE_URL=sqlite+aiosqlite:///D:/rag/linkrag_sdd/src/backend/linkrag.db
```

或兼容模式：

```dotenv
SQLITE_DATABASE=linkrag.db
```

### 2.2 PostgreSQL

```dotenv
DATABASE_URL=postgresql+asyncpg://postgres:password@127.0.0.1:5432/linkrag
```

---

## 3. 表结构同步（幂等）

在 `src/backend` 目录执行：

```powershell
python migrate_db.py
```

说明：
- 会按 `database/models.py` 创建缺失表。
- 会补齐历史版本缺失列（如 `knowledge_bases.es_id`、`chat_sessions.description`）。
- 可重复执行（幂等）。

---

## 4. SQLite 迁移到 PostgreSQL

在 `src/backend` 目录执行以下步骤：

1. 安装依赖：

```powershell
pip install -r requirements.txt
```

2. 设置环境变量：

```powershell
$env:SQLITE_DATABASE = "linkrag.db"
$env:DATABASE_URL = "postgresql+asyncpg://postgres:password@127.0.0.1:5432/linkrag"
```

3. 执行迁移脚本：

```powershell
python migrate_sqlite_to_postgres.py
```

4. 切换运行环境到 PostgreSQL：

```dotenv
DATABASE_URL=postgresql+asyncpg://postgres:password@127.0.0.1:5432/linkrag
```

5. 启动后端前执行一次结构同步：

```powershell
python migrate_db.py
```

---

## 5. 团队协作建议

- 开发环境可继续 SQLite，测试/预发/生产统一 PostgreSQL。
- 变更模型后，必须：
  1) 更新迁移逻辑（`migrate_db.py` 或连接层补列）
  2) 更新文档（本文件与 `数据库设计.md`）
- 禁止将本地 `.db` 文件提交到仓库。
