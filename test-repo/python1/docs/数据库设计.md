# 数据库设计文档

本文档描述 RAG 知识库系统后端数据库表结构、字段说明与实体关系。实现位于 `src/backend/database/models.py`，支持 SQLite 与 PostgreSQL。

---

## 1. 技术栈

| 项目 | 说明 |
|------|------|
| 数据库 | SQLite / PostgreSQL |
| ORM | SQLAlchemy 2.0（异步：aiosqlite / asyncpg） |
| 迁移 | 手写脚本（如 `migrate_db.py`、`migrate_sqlite_to_postgres.py`） |

---

## 2. 表清单

| 表名 | 说明 |
|------|------|
| users | 用户 |
| permissions | 权限（角色-资源-动作） |
| knowledge_bases | 知识库 |
| documents | 文档 |
| doc_virtual_folders | 文档虚拟文件夹 |
| doc_folder_assignments | 文档与文件夹关联 |
| document_chunks | 文档分块（向量化前内容） |
| chat_sessions | 对话会话 |
| chat_messages | 对话消息 |
| query_logs | 检索/问答日志 |
| system_logs | 系统日志 |
| system_config | 系统配置（key-value） |

---

## 3. 表结构详情

### 3.1 users（用户表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL, INDEX | 用户名 |
| password_hash | VARCHAR(128) | NOT NULL | 密码哈希 |
| email | VARCHAR(100) | NULL | 邮箱 |
| nickname | VARCHAR(50) | DEFAULT '用户' | 昵称 |
| avatar | VARCHAR(255) | NULL | 头像 URL |
| role | VARCHAR(20) | DEFAULT 'user' | 角色：admin=管理员，user=开发人员，guest=用户（鉴权中心） |
| is_active | BOOLEAN | DEFAULT TRUE | 是否启用 |
| last_login | DATETIME | NULL | 最后登录时间 |
| created_at | DATETIME | | 创建时间 |
| updated_at | DATETIME | | 更新时间 |

**索引**：idx_users_role, idx_users_active

---

### 3.2 permissions（权限表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| role | VARCHAR(20) | NOT NULL, INDEX | 角色 |
| resource | VARCHAR(50) | NOT NULL | 资源 |
| action | VARCHAR(20) | NOT NULL | 动作 |
| created_at | DATETIME | | 创建时间 |

**索引**：idx_permissions_role_resource

---

### 3.3 knowledge_bases（知识库表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| name | VARCHAR(100) | NOT NULL | 名称 |
| description | TEXT | NULL | 描述 |
| icon | VARCHAR(50) | DEFAULT '📚' | 图标（emoji） |
| color | VARCHAR(20) | DEFAULT '#667eea' | 主题色 |
| creator_id | INTEGER | FK(users.id), SET NULL | 创建者 |
| is_public | BOOLEAN | DEFAULT TRUE | 是否公开 |
| document_count | INTEGER | DEFAULT 0 | 文档数 |
| total_chunks | INTEGER | DEFAULT 0 | 分块总数 |
| embedding_model | VARCHAR(100) | NULL | 嵌入模型 |
| status | VARCHAR(20) | DEFAULT 'active' | 状态：active / archived |
| es_id | VARCHAR(100) | NULL | ES 索引名，空则用 linkrag_kb_{id} |
| chunk_size | INTEGER | DEFAULT 4096 | 分块大小 |
| chunk_overlap | INTEGER | DEFAULT 256 | 分块重叠 |
| chunk_strategy | VARCHAR(50) | DEFAULT 'general' | 分块策略 |
| created_at | DATETIME | | 创建时间 |
| updated_at | DATETIME | | 更新时间 |

**索引**：idx_kb_creator, idx_kb_status

---

### 3.4 documents（文档表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| knowledge_base_id | INTEGER | FK(knowledge_bases.id), CASCADE | 所属知识库 |
| filename | VARCHAR(255) | NOT NULL | 文件名 |
| file_path | VARCHAR(500) | NOT NULL | 存储路径 |
| file_size | INTEGER | DEFAULT 0 | 文件大小 |
| file_type | VARCHAR(50) | NULL | 文件类型 |
| content_hash | VARCHAR(64) | NULL, INDEX | 内容哈希 |
| tags | JSON | NULL | 标签 |
| status | VARCHAR(20) | DEFAULT 'pending' | pending / processing / completed / failed / uploaded |
| chunk_count | INTEGER | DEFAULT 0 | 分块数 |
| created_at | DATETIME | | 创建时间 |
| updated_at | DATETIME | | 更新时间 |

**索引**：idx_documents_kb, idx_documents_status, idx_documents_created

---

### 3.5 doc_virtual_folders（文档虚拟文件夹）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(64) | PK | 文件夹 ID |
| name | VARCHAR(255) | NOT NULL | 名称 |
| parent_key | VARCHAR(64) | NOT NULL, INDEX | 父级：kb-{id} 或 vf-{id} |
| kb_id | INTEGER | FK(knowledge_bases.id), CASCADE | 所属知识库 |
| created_at | DATETIME | | 创建时间 |

**索引**：idx_doc_vf_parent_kb

---

### 3.6 doc_folder_assignments（文档与文件夹关联）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| document_id | INTEGER | PK, FK(documents.id), CASCADE | 文档 ID |
| parent_key | VARCHAR(64) | NOT NULL, INDEX | 归属：vf-{id} 或 kb-{id} |

无记录表示文档在知识库根下。

**索引**：idx_doc_fa_parent

---

### 3.7 document_chunks（文档分块）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| document_id | INTEGER | FK(documents.id), CASCADE | 文档 ID |
| chunk_index | INTEGER | NOT NULL | 分块序号 |
| content | TEXT | NOT NULL | 文本内容 |
| token_count | INTEGER | DEFAULT 0 | token 数 |
| embedding_id | VARCHAR(100) | NULL | 向量库中的 ID |
| extra_data | JSON | NULL | 扩展数据 |
| created_at | DATETIME | | 创建时间 |

**索引**：idx_chunks_document, idx_chunks_embedding

---

### 3.8 chat_sessions（对话会话）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| session_key | VARCHAR(100) | UNIQUE, NOT NULL, INDEX | 会话 key |

| model_config | JSON | NULL | 模型配置 |
| summary | TEXT | NULL | 摘要 |
| message_count | INTEGER | DEFAULT 0 | 消息数 |
| total_tokens | INTEGER | DEFAULT 0 | 总 token |
| created_at | DATETIME | | 创建时间 |
| updated_at | DATETIME | | 更新时间 |

**索引**：idx_sessions_user, idx_sessions_kb, idx_sessions_updated

---

---

## 4. 实体关系（简要）

```
users ──┬──< knowledge_bases (creator_id)
        └──< chat_sessions (user_id)

knowledge_bases ──┬──< documents
                 ├──< chat_sessions
                 ├──< doc_virtual_folders
                 └──< query_logs

documents ──┬──< document_chunks
            └──< doc_folder_assignments

chat_sessions ───< chat_messages
```

---

## 5. 迁移与初始化

- **初始化**：`database/connection.py` 中 `init_db()` 调用 `Base.metadata.create_all` 建表。
- **迁移脚本**：`migrate_db.py`、`migrate_add_es_id.py`、`migrate_add_model_config.py` 等。
- **建议**：表结构变更时同步更新本文档；可考虑引入 Alembic 统一迁移管理。
