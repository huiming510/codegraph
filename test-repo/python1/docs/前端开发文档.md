# 前端开发文档

本文档说明 **src/frontend** 的环境、目录、启动、联调与开发规约。与 [开发框架说明](./开发框架说明.md) 配合使用。

---

## 1. 技术栈与环境

| 项目 | 说明 |
|------|------|
| 框架 | Vue 3 |
| 构建 | Vite 5 |
| 路由 | Vue Router 4 |
| 状态 | Pinia（含持久化） |
| UI | Ant Design Vue 4 |
| 请求 | Axios（封装在 src/api/index.js） |
| 样式 | Sass/SCSS |
| Node | >= 18.0.0（见 package.json engines） |

### 1.1 依赖安装

```bash
cd src/frontend
npm install
# 或
pnpm install
```

若路径变更或报错，可先删除 `node_modules` 再重新安装。

## 2. 目录结构

```
src/frontend/
├── index.html
├── vite.config.js       # Vite 配置；代理 /api -> 后端
├── package.json
├── .env.development     # 开发环境变量
├── .env.production      # 生产环境变量
├── public/              # 静态资源（不经过 Vite 处理）
└── src/
    ├── main.js          # 应用入口
    ├── App.vue          # 根组件
    ├── api/             # 请求封装与按模块拆分
    │   ├── index.js     # axios 实例（baseURL、拦截器、{ code, data, msg }）
    │   ├── helper/      # 请求相关工具（如 checkResStatus）
    │   └── modules/     # 按业务：knowledgeBase、document、login、rag 等
    ├── config/          # 请求常量、NProgress 等
    ├── constants/       # 路由白名单、应用常量
    ├── router/          # 路由定义与守卫、动态路由
    ├── stores/          # Pinia：auth、user、global 等
    ├── views/           # 页面级组件（知识库、文档、对话、系统等）
    ├── layouts/         # 布局（含侧栏、顶栏）
    ├── components/      # 通用组件（Loading、ErrorPages、SearchBar 等）
    ├── assets/           # 图片、字体、svg 等
    ├── styles/          # 全局样式（reset、common）
    ├── utils/            # 工具函数
    ├── directives/      # 自定义指令
    ├── hooks/            # 组合式函数
    └── language/        # 国际化
```

## 3. 启动与运行

```bash
cd src/frontend
npm run dev
# 或
pnpm dev
```

- 默认端口：**5174**（可在 `.env.development` 中修改 `VITE_APP_PORT`）
- 访问：<http://localhost:5174>
- 代理：请求 `/api` 会转发到 `vite.config.js` 中配置的 target（需与后端端口一致）

---

## 4. 环境变量

| 变量 | 说明 | 开发默认 |
|------|------|----------|
| VITE_APP_TITLE | 应用标题 | RAG 知识库 |
| VITE_APP_API_BASE_URL | 接口基础路径 | /api |
| VITE_APP_PORT | 开发服务器端口 | 5174 |
| VITE_APP_ROUTER_MODE | hash / history | hash |
| VITE_OPEN | 启动后是否打开浏览器 | true |

开发时使用 `.env.development`，生产构建使用 `.env.production`。修改后需重启 dev server。

---

## 5. 与后端联调

1. 先启动后端（见 [后端开发文档](./后端开发文档.md)），确认端口（如 8000）。
2. 若后端不是 8000，修改 `src/frontend/vite.config.js` 中 `server.proxy["/api"].target` 为实际地址（如 `http://localhost:8000`）。
3. 前端请求带 Cookie/Token：axios 已配置 `withCredentials: true`，登录后 token 存 store，拦截器自动加 `Authorization: Bearer <token>`；401 时跳转登录。

---

## 6. 开发规约摘要

- **请求**：统一通过 `import request from "@/api"`，不在其它地方新建 axios 实例；新接口写在 `api/modules/xxx.js`，路径与后端一致（不含 `/api`，baseURL 已带）。
- **页面**：新页面放在 `views/` 下；若需菜单，在动态路由或静态路由中配置，组件路径与 `views` 目录对应。
- **状态**：使用 Pinia，不混用 Vuex。
- 完整规约见 **.cursor/rules/frontend-conventions.mdc**。
