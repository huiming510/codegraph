# 后端开发文档

本文档说明 **src/backend** 的环境、目录、启动、调试与开发规约。与 [开发框架说明](./开发框架说明.md) 配合使用。

---

## 1. 技术栈与环境

| 项目 | 说明 |
|------|------|
| 语言 | Python 3.12+ |
| 框架 | FastAPI |
| 数据库 | SQLite / PostgreSQL（SQLAlchemy 2.0 AsyncSession） |
| 认证 | JWT（routers/deps.py） |
| 可选 | Elasticsearch（检索/解析入库）、PyYAML（解析 worker 读配置） |

### 1.1 依赖安装

在项目根目录使用同一虚拟环境（推荐 `.venv`）：

```powershell
# 创建并激活 venv（若尚未创建）
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1

# 安装后端依赖
pip install -r src/backend/requirements.txt
pip install PyYAML
```

如需对接 LLM/Embedding，按需安装对应 SDK（见 requirements 注释）。

## 3. 启动与运行

必须设置 **PYTHONPATH** 包含项目 **src** 目录，并在 **src/backend** 下启动：

```powershell
# 项目根目录
$env:PYTHONPATH = "d:\ai-rag\linkrag\src"   # 或 (Get-Location).Path + "\src"
cd src\backend
..\..\.venv\Scripts\python.exe -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

- 接口文档（Swagger）：<http://localhost:8000/docs>
- 根路径：<http://localhost:8000/> 返回 API 名称与 LLM 提供商

## 4. 配置与环境变量

- 配置文件：`config.py` 使用 Pydantic Settings，从环境变量或 `.env` 读取。
- 示例：复制 `src/backend/.env.example` 为 `.env`，按需填写：
  - `JWT_SECRET`、`DATABASE_URL`（或 `SQLITE_DATABASE`）、CORS、Elasticsearch、LLM/Embedding 等。
- 敏感信息不要提交到仓库，`.env` 已在 .gitignore 中。

### 4.1 数据库连接建议

- 推荐统一使用 `DATABASE_URL`：
  - SQLite：`sqlite+aiosqlite:///D:/rag/linkrag_sdd/src/backend/linkrag.db`
  - PostgreSQL：`postgresql+asyncpg://postgres:password@127.0.0.1:5432/linkrag`
- 未设置 `DATABASE_URL` 时，后端回退到 `SQLITE_DATABASE`（默认 `linkrag.db`）。

### 4.2 linkrag.db 管理约定

- `linkrag.db` 属于本地运行时数据，已被 `.gitignore` 中 `*.db` 规则忽略，不纳入 Git 版本管理。
- 结构变更通过迁移脚本与文档管理，不通过提交 `db` 文件。
- 启动前可执行：`python src/backend/migrate_db.py`，确保缺失表/列补齐。

### 4.3 SQLite 迁移到 PostgreSQL

1. 安装依赖：`pip install -r src/backend/requirements.txt`
2. 准备目标库：创建 PostgreSQL 数据库（如 `linkrag`）
3. 在 `src/backend` 目录设置环境变量：
   - `SQLITE_DATABASE=linkrag.db`（或绝对路径）
   - `DATABASE_URL=postgresql+asyncpg://postgres:password@127.0.0.1:5432/linkrag`
4. 执行迁移：`python migrate_sqlite_to_postgres.py`
5. 启动后端前，将运行环境固定为 PostgreSQL `DATABASE_URL`

## 5. 开发规约摘要

- **响应格式**：统一 `{ code, data, msg }`，成功使用 `success_response(data=...)`，失败用 `HTTPException`。
- **认证**：需要登录用 `Depends(require_user)`，需要管理员用 `Depends(require_admin)`。
- **新增接口**：在 `routers/` 下新增或扩展模块，在 `main.py` 中 `include_router`；业务逻辑尽量放在 `database/crud.py` 或 `server/`。
- 完整规约见 **.cursor/rules/backend-conventions.mdc**。

## 6. 常见问题

| 问题 | 处理 |
|------|------|
| `ModuleNotFoundError: No module named 'config'` | 未在 **src/backend** 下启动或未设置 PYTHONPATH 为 **src**。 |
| 解析 worker 未启动（yaml 等） | 安装 `PyYAML`，并保证 PYTHONPATH 包含 src（server 会引用 preprocess、utils 等）。 |
| 端口被占用 | 修改 uvicorn 的 `--port`，并同步修改前端 vite 代理 target。 |

更多见 [开发框架说明](./开发框架说明.md) 与 [API 文档](./API文档.md)。
