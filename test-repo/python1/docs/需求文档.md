## 1. 需求概述

### 1.1 背景与目标

- 背景：简述业务或技术背景。
- 目标：本版本要达成的核心目标（可多条）。

### 1.2 范围说明

- **在范围内**：本版本包含的功能/模块列表。
- **不在范围内**：明确不做的内容，避免理解偏差。

---

## 3. 非功能需求（可选）

| 类型 | 描述 |
|------|------|
| 性能 | 如：列表首屏 < 2s，接口超时时间等 |
| 安全 | 如：敏感接口需登录、密码加密等 |
| 兼容 | 如：浏览器、分辨率、后端版本等 |

---

## 4. 接口与依赖

- **依赖后端接口**：列出本需求涉及的主要 API（可引用 [API 文档](./API文档.md) 或 Swagger）。
- **依赖第三方**：如 LLM、向量库、对象存储等。
- **前后端约定**：若本需求有新增接口，说明路径与响应格式（与现有 `{ code, data, msg }` 保持一致）。

---

## 5. 验收与发布

- **验收条件**：以上功能点与非功能点均通过测试。
- **发布说明**：发布顺序（如先后端后前端）、配置项、数据迁移（若有）。
- **回滚方案**：简要说明回滚步骤或引用运维文档。
