# 知识库应用问答 - 修改总结

本文档汇总**知识库应用问答**（智能问答 / 助手与对话）相关的前端界面、后端服务接口、数据库及访问服务、页面/功能跳转逻辑（入参）。

---

## 1. 前端界面

### 1.1 页面位置与入口

| 项目 | 说明 |
|------|------|
| **列表页路径** | `src/frontend/src/views/chat/list/index.vue` |
| **详情页路径** | `src/frontend/src/views/chat/detail/index.vue` |
| **路由 path** | `/chat/list`（助手列表）、`/chat/detail`（对话详情） |
| **菜单归属** | 智能问答（admin、user、guest 均可见，由后端 `GET /api/auth/menu` 按角色过滤） |

### 1.2 页面功能

#### 1.2.1 助手列表页（/chat/list）

- **页面标题**：学习业务知识，解决你的疑惑。
- **统计**：显示助手数量（如「N 个助手」）。
- **助手卡片列表**：每张卡片展示助手名称、描述、知识库数、对话数、更新时间。
- **操作**：新建助手、编辑、删除；点击卡片进入该助手下的对话详情。
- **新建/编辑弹窗**：助手名称、头像、角色（description）、知识库（多选）、检索配置（search_options：top_k、threshold、search_method）、生成配置（generate_options：temperature、top_p）。编辑时通过 `getChatAppDetail` 拉取最新配置，保证展示数据与数据库一致。

#### 1.2.2 对话详情页（/chat/detail）

- **左侧**：会话列表（当前助手下所有对话），支持新建对话、切换对话、删除对话。
- **右侧**：聊天主区域，包括：
  - 顶部：助手名称、在线标识、对话配置入口、知识库选择器、清空消息、删除对话。
  - 对话配置弹窗：角色（description）、search_options、generate_options；打开时先 `getChatAppDetail` 拉取最新配置，取消时还原为打开前的快照；保存后调用 `updateChatAppConfig` 持久化到助手。
  - 消息列表：欢迎语 + 快捷问题；或气泡消息（用户/AI），AI 消息可带参照文档卡片（references，展示文件名+工作表名，悬浮显示完整元信息）。
  - 输入框：支持 Enter 发送、Shift+Enter 换行；思考模式开关（UI 展示，当前版本未传给后端）。

## 2. 后端服务接口

### 2.1 路由模块

**文件**：`src/backend/routers/rag.py`  
**前缀**：`/api`（与其他 RAG 接口共用）  
**权限**：依赖 `Depends(get_current_user)`，支持已登录或匿名访问。

### 2.2 接口列表（知识库应用问答相关）

| 方法 | 路径 | 说明 | 主要入参 / Body |
|------|------|------------------|------------------|
| **助手管理** ||||
| POST | /api/chat-apps | 创建助手 | Body: `name`, `icon?`, `description?`, `kb_ids?`, `search_options?`, `generate_options?` 等 |
| GET | /api/chat-apps | 助手列表（仅 app_ 且无 parent；需登录，未登录返回空） | 无 |
| GET | /api/chat-apps/{session_key} | 助手详情 | Path: `session_key` |
| PUT | /api/chat-apps/{session_key} | 更新助手 | Body: 同创建 |
| PUT | /api/chat-apps/{session_key}/config | 仅更新对话配置 | Body: `description?`, `search_options?`, `generate_options?` |
| DELETE | /api/chat-apps/{session_key} | 删除助手或对话 | Path: `session_key`（app_ 时级联删除其下所有 conv_） |
| **对话管理** ||||
| GET | /api/chat-apps/{app_session_key}/conversations | 某助手下对话列表 | Path: `app_session_key` |
| POST | /api/chat-apps/{app_session_key}/conversations | 在助手下新建对话 | Path: `app_session_key` |
| **对话与历史** ||||
| POST | /api/chat/stream-external | 多轮流式对话代理 | Body: `session_id`, `utterance`, `index`, `app_session_key?`, `system_prompt?`（角色/system 提示词）, `search_options`, `generate_options` |
| GET | /api/chat/history/{session_id} | 会话历史 | Path: `session_id`；返回 `{ messages: [{ role, content, timestamp, references }] }`；权限：仅当前用户所属会话 |
| PUT | /api/chat/sessions/{session_key} | 更新会话/对话标题 | Path: `session_key`; Body: `{ "title": "..." }` |

### 2.3 流式对话说明

- 前端调用 `sendChatStream` 请求 `POST /api/chat/stream-external`，传入 `systemPrompt`（即 chatConfig.description，角色/system 提示词）；后端将 `system_prompt`、`search_options`、`generate_options` 等转发到外部多轮对话 API（`settings.chat_stream_url` + `/chat/stream`）。
- 会话不存在且提供 `app_session_key` 时，后端自动创建 conv_ 会话并关联到该助手。
- 用户消息与助手回复会持久化到 `chat_messages` 表；流结束后解析 SSE 中的 `delta` 与 `references`，写入助手消息（含 `extra_data.references`）；若外部 API 将 references 以「参考片段：[...]」形式追加到正文，会解析并剥离后分别存 content 与 extra_data。
- SSE 事件：`delta`（块内容）、`references`（参考片段 JSON）、`[DONE]`。

## 3. 数据库及访问服务

### 3.1 表：chat_sessions（对话会话）

**文档**：`docs/数据库设计.md` 第 3.8 节（本表有扩展，实际实现见 models.py）  
**模型**：`src/backend/database/models.py` 中 `ChatSession` 类

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| session_key | VARCHAR(100) | 会话 key，唯一；**app_** 开头为助手，**conv_** 开头为对话 |
| parent_session_key | VARCHAR(100) | 对话所属助手 key（conv_ 时非空；app_ 且无 parent 表示顶级助手） |
| title | VARCHAR(200) | 标题，默认「新对话」 |
| description | TEXT | 助手角色描述（app_ 时有效，对应「角色」配置） |
| user_id | INTEGER | 用户 ID，可空 |
| knowledge_base_id | INTEGER | 关联知识库，可空 |
| model_config | JSON | 模型配置（含 name、icon、kb_ids、search_options、generate_options 等） |
| summary | TEXT | 摘要，可空 |
| message_count | INTEGER | 消息数 |
| total_tokens | INTEGER | 总 token |
| created_at / updated_at | DATETIME | 创建/更新时间 |

**索引**：idx_sessions_user、idx_sessions_kb、**idx_sessions_parent**、idx_sessions_updated。

### 3.2 表：chat_messages（对话消息）

**文档**：`docs/数据库设计.md` 第 3.9 节  
**模型**：`src/backend/database/models.py` 中 `ChatMessage` 类

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| session_id | INTEGER | 会话 ID（FK chat_sessions，CASCADE 删除） |
| role | VARCHAR(20) | user / assistant |
| content | TEXT | 文本内容 |
| tokens_used | INTEGER | 消耗 token |
| model | VARCHAR(50) | 模型名，可空 |
| latency_ms | INTEGER | 延迟 ms，可空 |
| extra_data | JSON | 扩展数据（如 `references` 数组，供参照文档卡片展示） |
| created_at | DATETIME | 创建时间 |

**索引**：idx_messages_session、idx_messages_created。

**文件**：`src/backend/database/crud.py`

## 4. 页面 / 功能跳转逻辑（入参）

### 4.1 进入助手列表页

| 方式 | 说明 | 入参 |
|------|------|------|
| **菜单** | 登录后点击「智能问答」→ 进入 `/chat/list` | 无 path 参数 |
| **直接 URL** | 浏览器访问 `/chat/list` | 无 query 参数 |

### 4.2 进入对话详情页

| 方式 | 说明 | 入参 |
|------|------|------|
| **从列表点击卡片** | 点击某助手卡片 | `goToChat(session)` → `router.push({ path: '/chat/detail', query: { app_session_key: session.session_key } })` |
| **从列表新建后** | 创建助手后跳转 | query: `app_session_key` |
| **直接 URL** | 如 `#/chat/detail?app_session_key=app_xxx` | query: `app_session_key` 或 `session_key`（兼容） |

**路由 query 约定**：

- `app_session_key`（或 `session_key`）：当前助手 session key（app_ 开头），必填；缺省时重定向到 `/chat/list`。
- `conv_session_key`：从某对话入口进入时，指定初始选中的对话；可选。

### 4.3 页面内操作与入参

| 操作 | 触发方式 | 入参 / 说明 |
|------|----------|-------------|
| **加载助手列表** | 列表页 `onMounted` 调用 `loadApps()` | `GET /api/chat-apps` |
| **创建助手** | 点击「新建助手」→ 弹窗 → 保存 | Body: name、icon、description、kb_ids、search_options、generate_options 等 |
| **编辑助手** | 点击卡片「编辑」→ 弹窗 → 保存 | `updateChatApp(session.session_key, sessionForm)` |
| **删除助手** | 点击「删除」→ 二次确认 | `deleteChatApp(session.session_key)` |
| **进入对话** | 点击卡片 | `app_session_key=session.session_key` |
| **加载对话列表** | 详情页根据 `app_session_key` watch | `getAppConversations(app_session_key)` |
| **新建对话** | 详情页点击「新建」 | `createAppConversation(app_session_key)` → 更新 `curSession`；新建后设置 `skipNextSessionLoad` 避免 watcher 覆盖首条消息 |
| **切换对话** | 点击左侧会话项 | `curSession = key`；优先用 `messageHistory` 缓存，否则 `getChatHistory(sessionKey)` |
| **发送消息** | 输入框 Enter | `sendChatStream({ sessionId, utterance, index, appSessionKey, systemPrompt, searchOptions, generateOptions })` |
| **清空当前消息** | 更多 → 清空当前消息 | 仅清空前端 `messages`、`messageHistory`；**不调用后端**，数据库记录保留 |
| **删除当前对话** | 更多 → 删除当前对话 | `deleteChatSession(curSession)` → `DELETE /api/chat-apps/{session_key}` |
| **保存对话配置** | 配置弹窗 → 保存 | `updateChatAppConfig(app_session_key, { description, search_options, generate_options })` |
| **更新对话标题** | 首条用户消息发送后 | `updateSessionTitle(session_key, title)`，静默失败不提示 |

### 4.4 与外部依赖的关系

- **流式对话**：`/api/chat/stream-external` 转发到外部 API（`settings.chat_stream_url`，默认 `http://192.168.10.187:8000`），需确保外部服务可访问。
- **知识库检索**：对话时需选择知识库；若助手配置了 `kb_ids`，详情页知识库选择器仅展示该助手的知识库，否则展示全部。前端将 `kb_id` 解析为 ES 索引名（`kb.es_id` 或 `linkrag_kb_{id}`）传给流式接口的 `index` 参数。

## 5. 实现细节与注意事项

| 项目 | 说明 |
|------|------|
| **description 存储** | 助手角色描述既存于 `chat_sessions.description` 也冗余于 `model_config.description`；对话配置更新时二者同步 |
| **无对话时创建** | 进入详情页若助手下无对话，自动调用 `createAppConversation` 创建首个对话 |
| **发送前无对话** | 用户发送首条消息时若 `curSession` 为空，先 `createAppConversation` 再发流式请求 |
| **references 展示** | 参照文档卡片取 `metadata.file_name`、`metadata.sheet_name` 等；悬浮 tooltip 显示完整元信息 |
| **流式错误处理** | 连接类错误（connect/refused/timeout）时返回「服务器连接失败」提示 |

## 6. 小结

| 层级 | 内容 |
|------|------|
| **前端** | 列表页 `views/chat/list/index.vue`，详情页 `views/chat/detail/index.vue`；API 封装在 `chat.js`、`chatApp.js`；常量在 `constants/chatConfig.js`。路由 `/chat/list`、`/chat/detail`；详情页必带 `app_session_key`。 |
| **后端** | `routers/rag.py` 提供 chat-apps 系列接口（助手 CRUD、对话创建/列表、配置更新）、流式对话代理、历史、会话标题更新；消息持久化到 `chat_messages`。 |
| **DB 与访问** | 表 `chat_sessions`（含 `description`、`parent_session_key` 区分助手/对话）、`chat_messages`（含 `extra_data.references`）；CRUD 在 `database/crud.py`。 |
| **跳转与入参** | 列表页无入参；详情页依赖 `app_session_key`；可选 `conv_session_key` 指定初始对话；表格/列表内操作以 `session_key`、`session_id` 等作为接口入参。 |
