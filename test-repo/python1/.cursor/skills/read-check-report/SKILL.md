# Read Code Check Report

When the user asks to see the check report, fix issues from the check, or "如何处理检查结果", read the report and explain in a structured way.

## Where to read

1. **Summary report**: `reports/check-report.md` (generated by `.\scripts\check_all.ps1`). Contains timestamp, backend/frontend status (OK/FAIL), and "处理建议". If the report has "### 最近输出" sections, they contain the last lines of the failing tool output.
2. **Full backend log**: `reports/check-backend.log` (from `.\scripts\check_backend.ps1`). Contains py_compile and ruff check/format output.
3. **Full frontend log**: `reports/check-frontend.log` (from `.\scripts\check_frontend.ps1`). Contains ESLint, Stylelint, and Prettier output.

If `reports/` does not exist or the files are missing, tell the user to run the check first: `.\scripts\check_all.ps1` (from project root).

## How to present to the user

1. **Summary**: State overall result (全部通过 / 部分未通过) and per-part (后端 OK/FAIL, 前端 OK/FAIL).
2. **Backend failures**: From the log or report, list each issue with **file:line** (or file path) and the rule/message. Common sources:
   - **ruff check**: Lines like `src/backend/path/file.py:10:1: E501 line too long`. Suggest: fix the line or run `ruff check src/backend --fix` for auto-fixable rules.
   - **ruff format**: "Would reformat ...". Suggest: `ruff format src/backend`.
   - **py_compile**: Syntax error and file path. Suggest: fix the reported file and line.
3. **Frontend failures**: From the log or report, list each issue with **file:line** and rule. Common sources:
   - **ESLint**: Path + line:column + rule + message. Suggest: fix manually or run `cd src/frontend && npm.cmd run check:fix`.
   - **Stylelint**: File + line + rule. Suggest: same `check:fix` or fix the style.
   - **Prettier**: "Code style issues found in ...". Suggest: `npm.cmd run check:fix` or `npm.cmd run lint:prettier`.
4. **Action list**: End with a short "如何处理" list (e.g. run ruff format, run check:fix, edit file X at line Y).

## Commands to suggest

- Backend format: `ruff format src/backend`
- Backend lint fix: `ruff check src/backend --fix`
- Frontend fix: `cd src/frontend && npm.cmd run check:fix`
- Re-run check: `.\scripts\check_all.ps1`

## If report is empty or all OK

If the report shows 全部通过, tell the user "检查全部通过，无需处理." If the user asked how to fix, briefly remind them how to run the check and where the report is written (reports/check-report.md).
